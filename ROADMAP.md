# 🚀 AI Logo Studio — Roadmap

> Обновлено по итогам аудита (см. audit-report.md) и feature-spec.md.
> Старая версия этого документа предлагала путь через OpenAI/DALL-E —
> от него отказались: дорого по API-вызовам, непредсказуемо по качеству,
> и не вписывается в позиционирование "самый дешёвый brand kit для
> малого бизнеса" без подписки.

## 📊 Текущий статус

### ✅ Что реально работает
- **Frontend**: Next.js + Tailwind, формы и галерея результатов
- **Backend skeleton**: FastAPI, CORS, health-check, метрики
- **SSE progress tracking**: чистый, рабочий endpoint
- **БД**: модели `Job` и `LogoVariant` корректны структурно

### ❌ Что сломано (по аудиту)
- **Генератор логотипов**: три каскадных уровня, все с проблемами —
  hardcoded формы по 5 индустриям, нондетерминированный
  `random.uniform()` без seed, PNG сохраняется с расширением `.svg`,
  хардкодные тэглайны типа `"since 2025"` независимо от клиента
- **Kafka/Zookeeper/Confluent**: 3 контейнера ради одного топика с
  одним воркером — избыточная инфраструктура
- **LocalStack**: хак с заменой URL `localstack:4566` → `localhost`,
  сломается в продакшене
- **User-модель**: определена, но не используется нигде
- **Alembic**: миграций ноль, таблицы создаются не пойми как

## 🎯 План по фазам

### Фаза 1 — Рабочий генератор (приоритет №1)

Цель: заменить весь текущий SVG-генератор на детерминированный
шаблонный движок. Без LLM, без DALL-E, без Stable Diffusion — это
осознанное решение, не временная заглушка.

- [ ] Иконки: открытая библиотека с MIT-лицензией (Phosphor или Tabler)
- [ ] Маппинг business_type → 3-5 иконок на выбор (не одна на индустрию)
- [ ] 5-8 curated пар Google Fonts вместо Arial/Georgia/Impact
- [ ] Палитры по правилам цветовой теории вместо 3 хардкод-пресетов
- [ ] Реальное название бизнеса в композиции вместо хардкод-тэглайнов
- [ ] Детерминизм через seed на основе business_name (последовательно
  по всему модулю, не только в одной из веток как сейчас)
- [ ] 4 варианта на один запрос
- [ ] Unit-тесты на детерминизм и на различимость между индустриями

### Фаза 2 — Инфраструктура

- [ ] Убрать Kafka из backend (`core/kafka.py`, вызовы в
  `routes_generate.py`, `main.py`)
- [ ] Удалить `worker/` целиком — генерация переезжает в backend,
  вызывается синхронно (или через `BackgroundTasks`, если нужен
  прогресс-бар — но при синхронном движке счёт идёт на миллисекунды,
  прогресс-бар, скорее всего, не нужен)
- [ ] Cloudflare R2 вместо LocalStack, без URL-хаков
- [ ] Первая настоящая Alembic-миграция
- [ ] `docker-compose.yml` → 3 сервиса (backend, frontend, db) вместо
  текущих Kafka/Zookeeper/Kafdrop/LocalStack/Prometheus/Grafana
- [ ] Убрать неиспользуемую `User`-модель (вернётся в Фазе 5 вместе с
  личным кабинетом — делать сейчас рано)

### Фаза 3 — Оплата

- [ ] Stripe Checkout, гостевая покупка без регистрации
- [ ] Watermark на превью, снятие после оплаты
- [ ] Доставка результата: прямое скачивание после оплаты или email
- [ ] Разовый платёж (не подписка) — это основа позиционирования

### Фаза 4 — Полировка MVP

- [ ] Vectorizer как бесплатный инструмент (VTracer/WASM в Web Worker) —
  загрузка наброска логотипа, лид-магнит в воронку к платному brand kit
- [ ] Редактор вариантов: смена иконки/шрифта/палитры без повторной
  генерации с нуля
- [ ] Экспорт: SVG, PNG, простой PDF со сводкой ассетов

### Фаза 5 — После первых платежей (не раньше)

- [ ] Личный кабинет: регистрация, история сгенерированных kit'ов
- [ ] Сохранённые пресеты (любимые палитры/шрифты)
- [ ] Командные/агентские тарифы
- [ ] PDF-брендбук с правилами использования (отступы, запрещённые
  искажения) — премиальная фича
- [ ] API для агентств/интеграций

## 📈 Метрики успеха (пересмотрены)

1. **Детерминизм**: одинаковый business_name+prefs → одинаковый
  результат при повторном запросе
2. **Разнообразие**: разные business_type дают визуально разные
  иконки/композиции (не сводятся к одному дефолту, как сейчас)
3. **Скорость**: генерация < 1 сек (синхронный шаблонный движок, без
  внешних API-вызовов — старая метрика "< 30 сек" была рассчитана под
  LLM/DALL-E, теперь неактуальна)
4. **End-to-end флоу**: brief → варианты → оплата → скачивание работает
  без Kafka и без LocalStack

## 🚫 Сознательно отложено / не делаем в MVP

- LLM-анализ бренда (GPT-4/DALL-E/Stable Diffusion) — дорого по
  API-вызовам, непредсказуемо, противоречит позиционированию "дешевле
  всех"
- Анимированные и 3D-логотипы
- Обучение на предпочтениях пользователей, A/B тестирование
- Подписочная модель — только разовый платёж в MVP
